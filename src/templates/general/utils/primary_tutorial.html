Introduction

This stand-alone tutorial can be run alone and will walk you through process of creating and executing a study on this platform. For the purpose of this tutorial, we will use the MPC-GWAS workflow with the auto-configured option. That said, this same tutorial can be followed with any of the workflows.

The purpose of a GWAS study is to identify genetic variants that are associated with a trait of interest. In this tutorial, we will use a simulated dataset to demonstrate the process of running a GWAS study with multiple participants. This tutorial uses a simulated dataset of genotypes, phenotypes, and covariates.

Note: There is also a tutorial for the user-configured option. That is available on the sfkit CLI documentation website.

Creating a study

The first step is to create a study. To do this, go to the Studies page and click Create New Study. (In a real study, you could also choose to join someone else's study instead.)

You will now see a page that looks something like this:

You are given the option to choose among any of the 3 workflows and 2 configuration options. More details about these choices are available in the "Instructions" page. For this tutorial, you can leave the defaults and then click Confirm to continue.

On the next page, you will be asked to provide a name for your study. Pick any unique study title, and you can leave the descritption and study information blank. Be sure to check the "Demo Study" option. Then proceed by clicking Confirm again. You will now be taken a a page to set the "Shared Study Parameters". For the demo, you can leave all of the parameters as they are. In a real study, you would set them parameters according to your data and preferences. Click Submit to proceed.

Configuring your study

You should now see a page that looks something like this (if you instead see the main study page, click the "Configure Study" button to get to this page):

0. When running a real study, you would need to follow the instructions to set up your GCP account. For the sake of this demo, these steps are optional, and the default configuration will run the demo on a GCP machine controlled by us. If you would like to run the demo on your own GCP account, you can follow the instructions. When you are done, you can click Next.

1. For the GCP Project ID, either leave it as is to use our machines or enter whatever GCP project you chose in step 0. Either way, for the sake of the demo, you can ignore the data paths. In a real study, you would need to upload your data to a bucket in your GCP account and indicate the paths to said data here. Click Save if you have made any changes. Once you are done with this step, you can click Next.

2. If you are using your own GCP project, please run the given command in your GCP cloud shell. This command will give the website permissions to set up the protocol for your study in your GCP project. Otherwise, you don't need to do anything here. Once you are done, you can click Next.

3. For the sake of this demo, you can leave the VM size as it is. You can click Done to proceed.

Running your study

You should now see a page that looks something like this:

This is your main study page. Here you can see the status of your study, and you can download the results when the study is complete. You can also click the "Configure Study" button to go back to the configuration page. This is also where you can view the study participants and add more participants to your study. When all partiicpants initiate their study, the joint study will begin. For this demo, you are the only real participant.

Click the Begin MPCGWAS Workflow button to begin your study. The study should take about 1/2 hour to complete, and status updates will be visible on this page. Feel free to leave this page and come back. When the study is complete, a link to download the results should appear.

When the study is complete, you can click the "Download results" link to download the results file with the association statistics.
